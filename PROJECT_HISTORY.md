## 12. Исправление мобильного меню (3 марта 2026)

**Проблема:** бургер-меню было прозрачным при прокрутке — контент страницы просвечивал сквозь навигацию.

**Причина:** CSS `transform` на анимированных элементах (карточки фич, статистика) создавал новые stacking context'ы, которые конкурировали с z-index хедера.

**Решение:** при открытии бургера JS переносит `<nav>` из `<header>` в `<body>`, обходя все stacking context'ы. При закрытии — возвращает обратно.

Затронутые файлы:
- `js/components/header.js` — логика перемещения nav
- `css/components.css` — z-index 9999 для header и burger
- `css/home.css` — убран z-index: 1 из секций
